# Архитектурные требования

Это живой документ с архитектурными требованиями к новому клиентскому веб-приложению Food Fleet.

Для получения дополнительной информации ознакомьтесь с [Проектной спецификацией](https://github.com/maks88sgt/food-fleet/blob/main/project-spec.md).

## Бизнес-цели

| Стейкхолдер                  | Цель                                                                                   | Контекст                                                                                         |
| ---------------------------- | -------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------ |
| Влад (CEO)                   | Увеличить количество клиентских заказов на 100% за один год                            | Как новый игрок на рынке, FoodFleet должен привлечь клиентов с помощью своего нового приложения. |
| Света (CTO)                  | Улучшить скорость команды и цикл разработки на 25%                                     | Новая архитектура должна позволить разработчикам быстрее поставлять фичи без потери качества.    |
| Дмитрий (Frontend Developer) | Доставлять код в продакшен уверенно, не опасаясь сломать фичи коллег                   | Текущая «большая куча грязи» делает трудно прогнозируемым влияние изменения кода.                |
| Илья (Customer Persona)      | Заказывать вкусную еду с телефона или компьютера и получать доставку как можно быстрее | Илья голоден и хочет съесть тарелку тако прямо сейчас.                                           |

## Ограничения

| Ограничение                                                                         | Контекст                                                                                                                 |
| ----------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------ |
| [Техническое] Должно быть развёрнуто в инфраструктуре Yandex Cloud                  | Наша DevOps-команда поддерживает и мониторит только сервисы Yandex Cloud.                                                |
| [Техническое] Должно быть адаптивным и полнофункциональным на мобильных устройствах | Более половины трафика приходит с мобильных, а нативное мобильное приложение будет готово не раньше чем через 6 месяцев. |
| [Бизнес] Должно быть выпущено в продакшен к марту 2026 года (через 4 месяца)        | Food Fleet планирует крупный маркетинговый запуск в феврале, и этот продукт — ключевой элемент стратегии.                |

## Атрибуты качества

| Атрибут качества                                  | Сценарий                                                                                                    | Приоритет |
| ------------------------------------------------- | ----------------------------------------------------------------------------------------------------------- | --------- |
| Performance (Производительность)                  | Пользователь на мобильном устройстве с 4G-соединением может загрузить приложение за ≤ 5 секунд.             | Высокий   |
| Scalability (Масштабируемость)                    | Кодовая база должна быть модульной, чтобы растущее число фронтенд-разработчиков могли работать параллельно. | Высокий   |
| Accessibility (Доступность)                       | Приложение должно соответствовать стандартам WCAG 2.2.                                                      | Средний   |
| Performance (Производительность)                  | Обновления в реальном времени должны доставляться всем подключённым клиентам в течение ≤ 15 секунд.         | Низкий    |
| Deployability (Удобство и быстрота развертывания) | Изменения в коде должны быть задеплоены в продакшен в течение ≤ 10 минут с начала релиза.                   | Низкий    |

## Другие влияющие факторы

- Сейчас фронтенд-команда состоит из 4 разработчиков, но ожидается рост в 3 раза в течение года.
- Каждый фронтенд-разработчик имеет опыт работы с React и Next.js; некоторые также знакомы с Vue.js и Laravel.
- Не все в команде уверенно владеют TypeScript.
