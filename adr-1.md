# ADR 1: Использование Next.js для веб‑приложения клиентов FoodFleed

## Статус

Принято

## Контекст

При подготовке к разработке нового веб‑приложения для клиентов FoodFleed нам необходимо принять ключевое решение: какой фреймворк для пользовательского интерфейса выбрать и какая инфраструктура потребуется для его развёртывания.

Цель — создать надёжное, масштабируемое и производительное веб‑приложение, которое эффективно удовлетворит потребности наших клиентов. Учитывая сжатые сроки проекта (4 месяца), мы должны стратегически подойти к выбору технологического стека, чтобы обеспечить быструю разработку и высокое качество результата.

## Решение

Мы приняли решение использовать Next.js в качестве фреймворка для разработки нового веб‑приложения клиентов FoodFleed и разместить его на собственной инфраструктуре AWS с использованием SST (Serverless Stack).

## Последствия

### Положительные последствия

- **Быстрая разработка.** Поскольку все члены команды уже знакомы с Next.js, мы можем использовать этот опыт для ускорения процесса разработки. Это поможет уложиться в сжатые сроки без ущерба для качества.
- **Функционально насыщенный фреймворк.** Next.js предоставляет ряд возможностей, полезных для современной веб‑разработки, включая серверный рендеринг, генерацию статических сайтов и API‑маршруты. Эти функции позволят нам создать производительное приложение с хорошей SEO‑оптимизацией.
- **Гибкость развёртывания.** Хотя мы не можем использовать Vercel для развёртывания, наш прототип с командой DevOps на базе Yandex Cloud показал, что мы успешно можем размещать приложения Next.js на собственной инфраструктуре. Это гарантирует использование возможностей Next.js без привязки к конкретному хостинг‑провайдеру.
- **Сообщество и экосистема.** Next.js имеет сильное сообщество и развитую экосистему, предоставляя нам богатый набор ресурсов, плагинов и интеграций с третьими сторонами, которые могут улучшить процесс разработки и расширить возможности приложения.

### Отрицательные последствия

- **Начальная настройка и конфигурация.** Развёртывание Next.js на собственной инфраструктуре YandexCloud требует этапа начальной настройки и конфигурации. Хотя прототип подтвердил реализуемость, этот процесс всё же потребует координации с командой DevOps и определённых начальных усилий.
- **Кривая обучения для кастомного развёртывания.** Команде потребуется время, чтобы ознакомиться с особенностями развёртывания Next.js в YandexCloud. Это может занять некоторое время на старте, хотя существенно не повлияет на общий график разработки.
- **Зависимость от внутренней инфраструктуры.** Выбирая развёртывание на собственной инфраструктуре, мы берём на себя полную ответственность за управление, масштабирование и обслуживание среды развёртывания. Это требует надёжной и слаженно работающей команды DevOps для обеспечения бесперебойной работы.

В целом решение об использовании Next.js хорошо согласуется с нашими текущими возможностями, ресурсами и требованиями проекта, позволяя нам выпустить качественное веб‑приложение для клиентов в установленные сроки.
